# ダッシュボード カード並び替え機能 設計書

## 目的

ダッシュボードのサービスカード一覧を、ユーザーが任意の基準で並び替えできるようにする。
現状は「支払いが近い順」で固定されており、登録順や金額順での閲覧ができない。

## 並び替えの種類

| キー | ラベル | ソートロジック |
|---|---|---|
| `billing` | 支払いが近い順 | `nextBillingFrom()` で算出した次回請求日の昇順 (デフォルト) |
| `added` | 追加順 | `createdAt` の降順 (新しいものが上) |
| `price-desc` | 高い順 | `price` の降順 (表示価格のまま、月額換算しない) |
| `price-asc` | 安い順 | `price` の昇順 (表示価格のまま、月額換算しない) |

「高い・安い順」は表示価格をそのまま比較する。
月額カード (例: 1,500円) と年額カード (例: 12,000円) はそれぞれの月額/年額セクション内で比較されるため、
セクションをまたいだ不自然な順序にはならない。

## 仕様

- 並び替えのドロップダウンは、カテゴリフィルターのピル行の **左端** に配置する
- セクション分け (月額払い / 年額払い) は **どの並び順でも常に維持** する。セクション内でソートが適用される
- 並び替えの状態は `ServiceGrid` 内の `useState` で保持する (URL searchParams には乗せない)
- カテゴリフィルター適用時のフラット表示 (`filter !== ALL`) でも同じソートを適用する
- デフォルトは `billing` (支払いが近い順)

## UI

- shadcn/ui の `Select` コンポーネントを使う (未導入の場合は `pnpm dlx shadcn@latest add select` を案内する)
- ピル行と同じ行に左端で並べる。レイアウトは `flex flex-wrap gap-2 items-center`
- ラベルは絵文字なし。プレースホルダー的なラベルは不要、選択中の値を表示する

## 影響範囲

### 変更ファイル

- `src/app/dashboard/page.tsx`
  - サーバー側で行っている `sorted` (次回請求日昇順) のロジックを削除し、`subs` をそのまま `ServiceGrid` に渡す
  - `monthlyTotal` / `yearlyTotal` の集計は `subs` ベースで継続 (ソート順に依存しないため影響なし)
- `src/components/dashboard/ServiceGrid.tsx`
  - ソート state (`sortKey`) を追加
  - `filtered` を計算したあとに `sortKey` で並び替えた配列を生成し、`SectionedGrid` / フラットグリッドの両方に渡す
  - ソート関数は同ファイル内のローカル関数として実装 (`sortSubs(subs, sortKey, today)`)
  - UI: ピル行の左端に `Select` を追加

### 変更しないもの

- DB スキーマ、Server Actions、`src/lib/billing.ts`
- カテゴリフィルターの挙動
- 月額/年額セクション分けの構造

## テスト

- `pnpm lint` / `pnpm tsc --noEmit` 通過
